# K-Contents trip

한국 드라마를 기반으로 한 체험형 여행 추천 에이전트 


---

##  프로젝트 소개 (OverView)

K-Contents trip 이란
한국 드라마를 기반으로 한 **체험형 여행 추천 서비스** 입니다.
단순 관광 정보를 제공하는 것이 아니라 
**드라마 속 인물로 하루를 살아보는 경험**을 목표로 합니다.
사용자는 드라마, 장면, 취향을 선택하면 
AI가 촬영지, 미식, 체험을 포함한 스토리 기반 여행 코스를 자동 생성합니다.

---
##  문제 정의 (Problem)

현재 관광 서비스는 다음과 같은 한계를 가지고 있습니다.
- 드라마 촬영지 / 음식 / 체험 정보가 분산되어 있음
- 여행이 단순 " 장소 방문 " 중심으로 구성됨
- 외국인 기준 정보 접근 및 예약 과정이 복잡함
- 미식 경험이 단순 맛집 추천에서 그침

---
## 해결 방식 (Solution)

본 서비스는 **드라마 서사 + 감정 + 장면** 을 기반으로 여행을 재구성합니다.
사용자는 
1. 드라마 선택
2. 장면 및 체험 선택
3. 여행 스타일 입력
이후 AI 가 다음 요소를 통합하여 코스를 생성합니다.
- 촬영지 및 포토 스팟
- 체험활동
- 지역 기반 한식 미식 코스
- 사용자 맞춤 일정

---

## 타겟 사용자 (Target Users)
- K-contents 를 통해 한국에 관심을 가진 외국인 관광객
- 단순 관광이 아니라 **스토리 기반 경험**을 원하는 사용자
- 개인 맞춤형 여행을 선호하는 20~30대
- 드라마 촬영지를 방문하고 싶은 국내 팬층

---

## 핵심 기능 (Key Features)
1️. 드라마 기반 탐색
- 인기 드라마 선택
  - 폭싹 속았수다
  - 오징어 게임

2. 장면 기반 체험 선택
- 예
  - 유체꽃밭 자전거
  - 해녀 체험
  - 골목 탐험
  - 포토 스팟
 
3. 캐릭터 기반 여행 코스 생성
사용자가 선택한 요소를 기반으로 **캐릭터의 하루** 형태의 코스 자동 생성
- 예
  - "순애로 살아보는 제주 하루"
  - "기훈이로 살아보는 하루"
단순 일정이 아니라 **감정 + 스토리 기반 경험 제공**

4. 미식 통합 추천
- 길거리 음식
- K-바비큐
- 모던 한식
- 전통 디저트
드라마 속 식사 장면과 연결된 **스토리형 미식 경험 제공**

5. 포토 스팟 & 장면 재현
- 촬영지 및 유사 장소 제공
- 장면 재현 가능 포인트 안내
- SNS 공유 유도

---

## 사용자 흐름 (User Flow)
1. 드라마 선택
2. 장면 및 체험 선택
3. 지역 추천 확인
4. 여행 스타일 입력
5. 음식 카테고리 선택
6. 캐릭터 기반 코스 선택
7. 사용자 수정 및 확정

---

## 기술 스택
**인프라(Infrastructure)**
- AWS : 클라우드 기반 인프라 및 배포 환경
- Docker : 개발 및 운영 환경 일관성 유지

**백엔드(Backend)**
- Python : 서비스 로직 및 API 구현

**AI/LLM**
- OpenAI API : LLM 기반 응답 생성 및 임베딩 처리
- LangChain: LLM 워크플로우 구성 및 체인 설계
- LangSmith: LLM 요청 추적 및 성능 모니터링(LLMOps)

**벡터 데이터 베이스(Vector DB)**
- Pinecone: 임베딩 기반 유사도 검색 및 추천 시스템 구현

**디자인**
- Figma : 디자인 확인 및 수정 작업을 통해 예상 화면 구축
